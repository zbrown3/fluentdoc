# Contributing to FluentDoc

Thank you for your interest in contributing to FluentDoc.

FluentDoc is an open-source, desktop-first platform for creating, documenting, and managing constructed languages.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting features
- Improving documentation
- Fixing issues
- Improving the desktop app experience
- Helping with language creation workflows

## Getting Started

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes
4. Test your changes locally
5. Submit a pull request

Example branch name:

feature/your-feature-name

## Running the Project Locally

### API

cd services/api  
./mvnw spring-boot:run

### Client

cd apps/client  
npm install  
npm start

## Pull Request Guidelines

Please keep pull requests focused and easy to review.

A good pull request should:

- Solve one clear problem
- Include a short explanation of the change
- Avoid unrelated formatting changes
- Update documentation when needed
- Include screenshots for UI changes when possible

## Code Style

Please follow the style already used in the project.

- Keep Angular components focused and readable
- Keep Spring Boot service logic separate from controller logic

## Issues

Before opening a new issue, please check if a similar issue already exists.

When reporting a bug, include:

- What happened
- What you expected to happen
- Steps to reproduce
- Screenshots or logs if useful
- Your operating system, if relevant

## Community Expectations

Be respectful, constructive, and patient.

FluentDoc is built for creators, linguists, writers, and worldbuilders. Contributions should support a welcoming and thoughtful creative community.
