Importing your dictionary

FluentDoc allows you to quickly populate your dictionary by uploading words in CSV, TSV, or JSON format. This guide explains supported file formats, required fields, and best practices for a smooth import process.


📂 Supported File Formats

  • ✅ CSV (Comma-Separated Values) – example.csv
  • ✅ TSV (Tab-Separated Values) – example.tsv
  • ✅ JSON (JavaScript Object Notation) – example.json

📄 Example File Structures


✅ CSV Example (dictionary.csv)
word,meaning,pos,pronunciation,notes
Ilnu,Tree,Common noun,ɪlnu,
Vorin,River,Common noun,voʊrɪn,
Fenlo,Sky,Common noun,fɛnloʊ,

✅ TSV Example (dictionary.tsv)
word	meaning	pos	pronunciation	notes
Ilnu	Tree	Common noun	ɪlnu
Vorin	River	Common noun	voʊrɪn
Fenlo	Sky	Common noun	fɛnloʊ

✅ JSON Example (dictionary.json)
{{ [
    {
      "word": "Ilnu",
      "meaning": "Tree",
      "pos": "Common noun",
      "pronunciation": "ɪlnu",
      "notes": null
    },
    {
      "word": "Vorin",
      "meaning": "River",
      "pos": "Common noun",
      "pronunciation": "voʊrɪn",
      "notes": null
    }
  ] | json }}

🔄 How to Upload Your File


  1. Go to the Dictionary Import section
  2. Drag & drop your file into the upload area OR click to browse
  3. Ensure your file format is correct (CSV, TSV, JSON)
  4. Click Import Words and wait for processing to complete
  5. If errors occur, review the message and adjust your file accordingly

🚨 Common Errors & Fixes


"Invalid File Type"

Cause: Unsupported format (e.g., Excel .xlsx)

Solution: Convert to CSV, TSV, or JSON


"Missing Required Fields"

Cause: word, meaning, or pos is missing

Solution: Ensure all required fields are included


"Incorrect JSON Format"

Cause: JSON syntax error (e.g., missing commas, incorrect structure)

Solution: Validate JSON structure using JSONLint


"Duplicate Words Detected"

Cause: Words already exist in your dictionary

Solution: Remove duplicates before importing


Best Practices


  • ✔ Use lowercase file extensions (e.g., dictionary.csv)
  • ✔ Ensure formatting is consistent across all rows
  • ✔ Avoid special characters in column names
  • ✔ Test with a small file before bulk importing
